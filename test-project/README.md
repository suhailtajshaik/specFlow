# test-project

API backend generated by [SpecFlow](https://github.com/suhailtajshaik/specflow).

## Quick Start

1. **Write rough specifications** in `requirements/`:
   - Add any .md files with your API ideas
   - Use bullet points, plain English, whatever works for you

2. **Transform into professional specs**:
   ```bash
   specflow prepare
   ```
   This analyzes your rough input and creates detailed .req.md and .contract.md files.

3. **Generate production code**:
   ```bash
   specflow generate
   ```
   Creates a complete TypeScript backend with Hono + Drizzle + PostgreSQL.

4. **Start development**:
   ```bash
   specflow dev
   ```

## Project Structure

```
requirements/
├── business/          # Business requirement files (.req.md)
├── technical/         # API contract files (.contract.md)  
└── schemas/           # Shared JSON schemas

generated/             # Generated backend code (after specflow generate)
├── src/
│   ├── routes/
│   ├── schemas/
│   ├── db/
│   └── server.ts
├── package.json
├── Dockerfile
└── docker-compose.yml
```

## Commands

- `specflow init` - Initialize new project
- `specflow prepare` - Transform rough specs into professional documentation
- `specflow generate` - Generate production TypeScript backend
- `specflow dev` - Generate + start development server  
- `specflow list` - List all API endpoints
- `specflow setup` - Check setup and installation help

## Configuration

Edit `specflow.config.yaml` to customize:
- LLM provider (Gemini, Claude, local LlamaCpp)
- Output directory and tech stack
- Database settings
- Docker configuration

## Need Help?

- Check `specflow setup` for installation issues
- Example specs: `specflow init --example`
- Documentation: [SpecFlow GitHub](https://github.com/suhailtajshaik/specflow)
